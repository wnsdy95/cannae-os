# Prompt Templates

## 0. 목적

이 문서는 군 문서 체계를 LLM 프롬프트로 바꾸기 위한 실전 템플릿 모음이다.

사용 원칙:

- 복잡한 작업은 OPORD로 시작한다.
- 아직 정보가 부족하면 WARNO로 준비시킨다.
- 중간 변경은 FRAGO로 기록한다.
- 실행 전 Backbrief를 받는다.
- 진행 중 SITREP과 CCIR로 보고한다.
- 끝나면 AAR로 학습한다.

## 1. WARNO: 사전 준비 지시

```text
WARNO: [작업명]

1. Situation
- 배경:
- 현재 알려진 것:
- 아직 모르는 것:

2. Mission
- 준비할 일:
- 준비 목적:
- 예상 본작업:

3. Execution
- 우선 수집할 자료:
- 확인할 제약:
- 생성할 준비 산출물:
- 하지 말아야 할 일:

4. Sustainment
- 사용할 도구:
- 사용할 자료:
- 시간/비용 제한:

5. Command and Signal
- 준비 결과 보고 형식:
- 즉시 보고 조건:
- 본 OPORD 발행 전 대기 여부:
```

## 2. OPORD: 본 실행 지시

```text
OPORD: [작업명]

1. Situation
- 배경:
- 작전 환경:
- 관련 자료:
- 제약:
- 정보 공백:
- 주요 위험:

2. Mission
- 누가:
- 무엇을:
- 언제까지:
- 어디에:
- 왜:
- 성공 조건:

3. Execution
- Commander's Intent:
  - 목적:
  - 최종 상태:
  - 핵심 우선순위:
  - 금지선:
- Concept of Operations:
  - 전체 접근:
  - 단계:
  - 주요 산출물:
- Tasks:
  - S2:
  - S3:
  - S4:
  - Red Team:
  - Executor:
  - Recorder:
- Coordinating Instructions:
  - 검증 기준:
  - 품질 기준:
  - 중단 조건:
  - 승인 필요 조건:

4. Sustainment
- 도구:
- 자료:
- 토큰/시간/비용 제한:
- 대체 수단:

5. Command and Signal
- 승인자:
- 정기 보고:
- 즉시 보고(CCIR):
- 변경명령(FRAGO) 처리 방식:
- 완료 보고 형식:
```

## 3. Backbrief: 실행 전 이해 확인

```text
BACKBRIEF

1. 내가 이해한 최종 목적:
2. 성공 조건:
3. 변형하면 안 되는 항목:
4. 내가 자율적으로 정할 수 있는 항목:
5. 승인받아야 하는 항목:
6. 실행 계획:
7. 검증 계획:
8. 예상 위험:
9. 즉시 보고할 조건:
10. 확인 질문:
```

## 4. FRAGO: 변경명령

```text
FRAGO [번호]: [작업명]

기준 문서:
변경 사유:
우선순위:

1. Situation
- 변경 없음 / 변경 내용:

2. Mission
- 변경 없음 / 변경 내용:

3. Execution
- 변경 없음 / 변경 내용:

4. Sustainment
- 변경 없음 / 변경 내용:

5. Command and Signal
- 변경 없음 / 변경 내용:

적용 시점:
기존 명령과 충돌 시 우선하는 항목:
승인 필요 여부:
```

## 5. SITREP: 상황보고

```text

1. 현재 상태:

3. 진행 중인 일:
4. 장애:
5. 위험:
6. 다음 행동:
7. 필요한 결정:
8. CCIR 발생 여부:
```

4. 현재 알려진 사실:
5. 불확실한 점:
6. 선택 가능한 방책:
7. 각 방책의 위험:
8. 필요한 승인:
9. 추천:
```

```
