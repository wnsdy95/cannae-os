# Code of Conduct

## Our Pledge

We want Cannae OS to be a focused, respectful, and technically serious project. Contributors should be able to discuss doctrine, safety, validation, and AI control without personal attacks or performative hostility.

## Expected Behavior

- Be direct, specific, and constructive.
- Critique ideas, code, documents, and assumptions rather than people.
- Make risks explicit when a proposal affects authority, release, security, or user safety.
- Respect different backgrounds and organizational cultures.
- Accept maintainer decisions about project scope.

## Unacceptable Behavior

- Harassment, threats, insults, or discriminatory language.
- Doxxing or sharing private information.
- Encouraging illegal, violent, coercive, or surveillance use.
- Repeatedly derailing reviews after a maintainer has closed a decision.
- Submitting intentionally misleading source claims or fabricated evidence.

## Enforcement

Maintainers may remove comments, close issues, reject PRs, restrict participation, or block users when behavior harms the project.

To report a concern, use the security/contact process in `SECURITY.md` if the report is sensitive. For ordinary conduct issues, open a private maintainer contact path if available or contact the repository owner.

## Scope

This code applies to project spaces, including issues, pull requests, discussions, and related community channels.
